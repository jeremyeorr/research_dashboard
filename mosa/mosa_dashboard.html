How to keep this dashboard in sync with REDCap via Power Query 
The browser cannot trigger Power Query. But this works end-to-end without manual steps:

  1. In Excel, open your xlsx. Data tab → Queries & Connections → right-click the REDCap
     query → Properties. Check:
        [x] Refresh every N minutes
        [x] Refresh data when opening the file
        [x] Enable background refresh
     Then turn on AutoSave (File → Info → or the AutoSave toggle) so the refreshed
     data is written to disk automatically.

  2. In this dashboard, click "Pick xlsx file" ONCE. Chrome/Edge remember the file
     handle, so "Refresh from file" re-reads whatever is currently saved to disk
     — no re-picking needed. Firefox/Safari don't support this API, so you'll
     need to re-pick the file each refresh.

  3. Turn on "Auto-refresh" and set a refresh interval in seconds. The dashboard
     will poll the file on disk. If Power Query + AutoSave are running in Excel,
     the dashboard will pick up fresh data automatically.

Expected xlsx structure: one sheet containing the REDCap flat export (records with
a record_id column and a redcap_event_name column). If
your Power Query lands in a table named differently, the dashboard picks the first
sheet that has both columns; otherwise it uses the first sheet.
